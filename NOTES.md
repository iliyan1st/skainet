# SKAINET – Project Notes

## What it is
Automated Bulgarian AI news portal. Fetches articles from 8 RSS sources, filters by AI/robotics keywords, translates to Bulgarian via Google Translate (free, no API key), outputs `articles.json`, and serves a static site on Vercel.

## URLs
- Live: https://skainet-vert.vercel.app/
- Google Search Console: https://search.google.com/search-console (verified via `google4065fee43f9851a8.html`)

## Tech stack
- Frontend: Vanilla HTML/JS, single-page (`index.html`, `article.html`)
- Data: `articles.json` (auto-generated, ~53 articles, ~500KB)
- Backend: Python (`scripts/fetch_articles.py`)
- Automation: GitHub Actions (`fetch-news.yml`) — runs every 6 hours
- Deploy: Vercel (auto-deploy on commit)

## Key files
| File | Purpose |
|------|---------|
| `scripts/fetch_articles.py` | RSS fetch → filter → translate → write `articles.json` + `sitemap.xml` |
| `index.html` | Homepage — card grid, search, hero |
| `article.html` | Article page — loads from `articles.json` by `?id=` param |
| `articles.json` | Article database (committed, ~53 items) |
| `sitemap.xml` | Auto-generated by the Python script |
| `vercel.json` | URL rewrite: `/article/:slug` → `/article.html?id=:slug` |

## Article URL routing
- Index links: `article.html?id={hash_id}` (e.g. `ab58d2520c4a`)
- Sitemap / canonical: `https://skainet-vert.vercel.app/article/{slug}`
- Vercel rewrite maps `/article/slug` → `?id=slug`
- `load()` in article.html finds by `a.id === id || a.slug === id` (handles both)

## Running locally
```bash
cd scripts
pip install feedparser requests trafilatura
python fetch_articles.py
```
Output: updates `articles.json` and `sitemap.xml` in project root.
